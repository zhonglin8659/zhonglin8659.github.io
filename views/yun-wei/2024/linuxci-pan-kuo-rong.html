    Linux磁盘扩容

    兜里有糖 2024/1/7 Linux

    # 系统盘动态扩容后空间扩充

    磁盘空间已经扩容，此时需要在系统中调整磁盘空间大小

    # 查看系统分区及大小

    fdisk -l
    1

    这里看到磁盘已经扩容至85.9G，实际挂载识别为39.7G

    # 开始添加分区

    fdisk /dev/sda
    1

    进入磁盘fdisk (util-linux 2.23.2)工具

    1. 输入n表示增加分区

    2. 输入p标识创建主分区

    3. 输入分区号（默认）

    4. 输入起始扇区、终止扇区（默认）

    5. 输入w进行保存

    # 修改新分区ID为系统8E

    通过上面的操纵，再次使用fdisk -l查看，可以看到新增加的分区/dev/sda3已经出现

    需要将红框处分区ID由83改成8E（LVM卷文件系统的ID）

    fdisk /dev/sda
    1

    1. 输入t

    2. 输入前面步骤的分区号3

    3. 输入l

    4. 输入8e

    5. 输入w进行保存

    修改后的信息如下：

    重启系统 reboot，这一步非常关键切记

    # 开始扩充分区

    使用lvs显示逻辑卷分区信息

    lvs
    1

    显示卷组信息

    vgdisplay
    1

    将物理硬盘分区初始化为物理卷，以便被 LVM 使用

    pvcreate /dev/sda3
    1

    扩展卷组：centos 是执行vgdisplay后的VG name对应的值，根据个人情况修改

    vgextend  centos /dev/sda3
    1

    再次显示卷组信息：可以看到Free PE从0变为40.00Gib，说明我们最多可以有 40GB 的扩充空间

    vgdisplay
    1

    扩充逻辑卷分区：centos/root 是根据第七部分一开头的lvs中显示的VG 和 LV内容进行的替换，根据个人情况修改

    lvextend -L+39.9G /dev/centos/root /dev/sda3
    1

    重设逻辑卷大小

    xfs_growfs /dev/centos/root
    1

    再次查看磁盘空间

    发现磁盘容量已经扩容完毕

    # 添加其他数据盘后空间扩充

    系统添加了一块32G的物理磁盘，此时需要在系统中初始化并挂载

    # 查看系统新增磁盘

    lsblk -f
    1

    sdb为新增的物理磁盘

    fdisk -l
    1

    /dev/sdb可以看到大小为32G

    # 开始添加分区

    我们使用系统自带的 fdisk 工具来划分和管理磁盘，该工具只能处理2TB以下的磁盘，大于2TB请使用 gdisk 工具

    fdisk /dev/sdb
    1

    输入 n 新建一个分区

    输入 p 表示新建一个主分区

    输入 1 默认分区号

    起始扇区 终止扇区（默认）

    输入 w 保存

    再次查看分区

    lsblk
    1

    # 分区初始化

    这里将分区初始化为ext4格式

    mkfs -t ext4 /dev/sdb1
    1

    lsblk -f
    1

    可以看到已经为 sdb1 分配了 UUID

    # 挂载分区

    命令挂载（临时挂载）

    1. 新建一个文件夹

      mkdir newdisk
      1

    2. 使用mount挂载

      mount /dev/sdb1 /newdisk/
      1

    3. 使用df -h查看挂载情况

      df -h
      1

    4. 使用umount卸载

      umount /newdisk/
      1

    配置挂载（永久挂载）

    1. 获取分区UUID

      lsblk -f
      1

    2. 编辑fstab文件

      第一个代表分区的UUID

      第二个代表要将该分区挂载的哪个地方

      第三个代表分区的类型，要与初始化时的类型保持一致

      第四个代表是否执行dump、是否执行检测

      添加如下：

      UUID=2d56bcfe-fe61-48a6-96db-0761b044c0fe /newdisk                   ext4     defaults        0 0
      1

      重启系统

    看到已经自动挂载成功
